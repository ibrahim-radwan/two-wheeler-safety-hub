ABOUT

No crumple zone, no airbag, half a second to react

Two-wheeled vehicles put riders in traffic with almost no reaction time and no physical protection. This project mounts a forward-facing RGB camera and a mmWave radar on a scooter, processes both streams entirely onboard, and turns the fused evidence into an interpretable, rider-facing risk warning — with no dependence on external compute or network infrastructure.

The system is split into four components, each in its own repository and linked from the pipeline and repos sections below.

THE PIPELINE

Four stages, in order

Each stage's output is the next stage's trusted input — reliable synchronization enables auditable annotation, and auditable annotation enables risk metrics a rider can actually trust.

01
HARDWARE SENSING PLATFORM

Mounted, powered, and self-contained

Camera, radar, and edge computer are rigidly mounted on the vehicle and run independently of any external network.

  • Intel RealSense D455 RGB camera, 640×480 @ 30 fps over USB 3.0
  • TI AWR2243 FMCW radar, 76–81 GHz, 3 Tx / 4 Rx over Ethernet/UDP
  • NVIDIA Jetson Orin runs acquisition, inference, recording, and the local web server
  • Portable power bank, onboard SSD, and a jetsonhotspot Wi-Fi access point
OUTPUT → timestamped RGB frames + raw radar recordings GitHub · Stage 01RoadSafetyOnTwoWheelers ↗
02
SYNCHRONIZED RECORDING & VoD PREP

Radar-led pairing, calibrated samples

Raw streams become calibrated, View-of-Delft-compatible samples ready for annotation.

  • The radar frame anchors each saved sample; the closest camera frame is attached with a logged clock offset
  • Seven-field radar rows: [X, Y, Z, I/P, v_r, v_rc, time_id]
  • Optional RANSAC-based ego-velocity compensation
  • 1-, 3-, or 5-scan temporal bundles with explicit scan ages
OUTPUT → calibrated, annotation-ready camera–radar samples GitHub · Stage 02FusionApp / vod_conversion ↗
03
RADAR–CAMERA ANNOTATION TOOL

One box, two views, human-reviewed

Every label is a single calibrated 3D box, checked in both radar and camera space before it's trusted.

  • One Box3D rendered as both a radar BEV footprint and an RGB wireframe
  • Manual placement plus RT-DETR/YOLO-assisted proposals
  • Identity-aware tracking with validation warnings and Fix actions
  • Export to internal JSON, KITTI/VoD label_2, and JSON/CSV
OUTPUT → auditable 3D object labels with temporal identities GitHub · Stage 03radar-camera-annotation-tool ↗
04
SAFETY DASHBOARD

Evidence you can trace, not a black box

Fused evidence becomes interpretable metrics and a risk level the rider can act on immediately.

  • Time-to-collision, forward distance, and brake-demand estimates
  • Four risk levels — Low, Medium, High, Critical — with explicit thresholds
  • Every object tagged as fused, radar-led, or camera-led evidence
  • Object table, risk timeline, and event log for full auditability
OUTPUT → explainable, time-aware hazard warnings GitHub · Stage 04twowheeler-safety-dashboard ↗
